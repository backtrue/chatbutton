# **產品需求文件 (PRD)：\[專案代號：ToldYou-Button\]**

### **版本：v1.1 (整合優化建議)**

### **產品核心定位**

**ToldYou-Button 是一款「為華語及亞洲市場打造的免費多平台聊天按鈕產生器」。**

**它用最簡單的在地化介面，讓任何人一分鐘內完成客服串接，並為我們的品牌資產 (thinkwithblack.com) 自然生成高品質的反向連結。**

## **1\. WHY (我們的存在理由：目標)**

### **1.1 Introduction (市場現況/問題)**

* **使用者問題：** 目前如 GetButton 的主流聊天按鈕工具，其介面與說明均為英文。這對於繁體中文使用者（特別是我們的學生、行銷人員、網站初學者）構成了顯著的「語言障礙」和「設定困惑」，降低了他們為網站配置客服工具的意願和效率。  
* **商業問題 (機會點)：** 我們目前被迫將自有流量（學生）導向第三方的英文服務，這不僅體驗不佳，更是「品牌資產的流失」。我們無法從這股流量中獲取任何回報（例如品牌曝光或 SEO 價值）。  
* **我們的機會：** 打造一款「中文優先」且「極度易用」的免費工具，填補在地化市場的缺口，並將這股流量轉化為我們自己的核心 SEO 資產。

#### **競品比較 (Competitive Landscape)**

| 工具 | 語系支援 | 使用難度 (P1) | 反向連結策略 | 價格 |
| :---- | :---- | :---- | :---- | :---- |
| **GetButton** | 英文 | 中 (進階設定多) | 無 (付費版可移除) | 付費 |
| **Chaty** | 英文 (後台) | 中 | 付費版可移除 | 付費 |
| **Elfsight** | 英文 | 高 (功能過於繁雜) | 付費版可移除 | 付費 |
| **ToldYou-Button** | **繁中 (P1) / 日 / 英 (P3)** | **極低 (P1)** | **是 (固定錨點文字)** | **永久免費** |

### **1.2 Objectives (我們的成功定義)**

* **商業目標 (P1-P3)：**  
  * **P1 核心目標：** 透過 P1 (繁中版) 的高安裝率，大規模地為 thinkwithblack.com 網站獲取高品質的 報數據 錨點文字反向連結 (Backlinks)。  
  * **P3 擴展目標：** 透過 P3 (日/英文版) 的在地化體驗，將反向連結策略擴展至全球，獲取 レポートデータ (日文) 和 Report Data (英文) 的錨點文字。  
* **產品目標：**  
  * **P1：** 成為繁體中文市場上「最簡單、零困惑」的多合一聊天按鈕生成器。  
  * **P2：** 將 P1 的一次性用戶，無痛轉移為可長期經營的 SaaS 會員，提升用戶黏著度。  
  * **P3：** 提供與繁中版同等易用性的日文與英文在地化版本。

### **1.3 核心 KPI (Key Performance Indicators)**

* **P1 (MVP)：** 30 天內達到 1,000 個獨立網站安裝 (installs)。  
* **P2 (SaaS 升級)：** P1 舊用戶登入 P2 後，30 天內的「P2 代碼升級轉換率」 ≥ 15%。  
* **P3 (全球擴張)：** P3 上線 60 天後，來自日文/英文版的新用戶佔比 ≥ 30%。

## **2\. WHO (我們的服務對象：人)**

### **2.1 Target Group (目標客群)**

* **P1 (主要)：** 繁體中文使用者。特別是您的學生、剛接觸 WordPress/Shopify 的電商賣家、行銷人員，他們需要一個「能用」且「易用」的免費工具。  
* **P3 (次要)：** 日本與英語系的網站管理員。他們會透過 P3 的在地化 SEO 和關鍵字廣告被吸引。

#### **市場規模估算 (TAM/SAM/SOM)**

* **TAM (潛在市場)：** 全球使用 WordPress / Shopify / Wix 等系統的網站 (數千萬)。  
* **SAM (可服務市場)：** 台灣使用 WordPress/Shopify 的活躍電商/形象網站（估算約 15-20 萬個）。  
* **SOM (P1 可獲取市場)：**  
  * **目標：** 假設 P1 階段能獲取 10% 的可服務市場 (SAM)。  
  * **量化價值：** 150,000 \* 10% \= 15,000 個高品質的「報數據」反向連結。

### **2.2 Stakeholders (利害關係人)**

* **Founder (您)：** 核心需求是「SEO 價值」。  
* **End-Users (您的學生/網站主)：** 核心需求是「免費」和「易用」。  
* **Development Team (開發團隊)：** 核心需求是「清晰的規格」。

## **3\. HOW (我們的體驗設計：故事)**

### **3.1 Persona / User Story 1: P1 新用戶 (中文學生)**

* **背景 (Who)：** 小明，剛上完您的行銷課程，正嘗試用 WordPress 建立自己的第一個形象網站。  
* **情境 (Where/When)：** 他想在網站右下角加上 LINE 和 FB Messenger 按鈕，以便客戶聯繫。他打開了您推薦的 GetButton，但被滿滿的英文術語 (Animation, Border, Shift, px) 搞得暈頭轉向。  
* **核心痛點 (Why)：** 「我只是想加個 LINE，為什麼這麼複雜？我怕設定錯誤。」  
* **Aha\! Moment (P1 關鍵時刻)：**「哇！這個『ToldYou-Button』產生器**全是中文的**。我只花了 1 分鐘就選好了 LINE、Messenger 和 WhatsApp，還選了顏色。輸入 Email 後，程式碼立刻就寄來了。**這也太簡單了吧！**」

### **3.2 Persona / User Story 2: P2 舊用戶 (P1 升級者)**

* **背景 (Who)：** 小明在 P1 上線 3 個月後，決定更換 WhatsApp 號碼。  
* **情境 (Where/When)：** 他回到 P1 工具頁，發現介面已經升級成 P2 會員系統。  
* **核心痛點 (Why)：** 「慘了，我只是想改個電話號碼，難道要我把所有設定（LINE, FB...）全部重新再輸入一次嗎？」  
* **Aha\! Moment (P2 關鍵時刻)：**「我用 Magic Link 登入後，**P1 的設定竟然全部自動匯入了！** 不但如此，它還提示我只要更換新版程式碼，就能**免費解鎖『歡迎訊息』的新功能**。這太棒了！」

### **3.3 Persona / User Story 3: P3 新用戶 (日本廣告主)**

* **背景 (Who)：** Kenji，一位在日本經營電商網站的廣告主。  
* **情境 (Where/When)：** 他在 Google 搜尋「チャットボタン 無料」(免費聊天按鈕)，並點擊了我們的 P3 關鍵字廣告，進入了 P1 的「日文版」工具頁。  
* **核心痛點 (Why)：** 「我之前試過一個外國工具，結果它在我的日文網站上放了一個奇怪的中文字連結，太不專業了。」  
* **Aha\! Moment (P3 關鍵時刻)：**「這個工具**介面全是日文的**，設定很直觀。最重要的是，它生成的反向連結是日文『レポートデータ』，這完全符合我的在地化需求。我願意使用它。」

## **4\. WHAT (我們的具體實現：規格)**

### **4.1 P1: 繁中 MVP (灘頭堡)**

* **產品設計 (UI/UX)：**  
  * **策略：** 「Clone & Simplify」—— 複製 GetButton 已驗證的「單頁設定」UX 流程。  
  * **移除 (Out of Scope)：** 砍掉 80% 的 P1 非必要功能，包括：AI Chatbot、Greeting Message、Live Preview、Animation, Border, Shadow, Button Style, Where to show...等所有進階設定。  
  * **在地化：** 介面 100% 繁體中文化，並提供針對新手的中文提示語。  
* **核心功能 (Features)：**  
  * **應用程式 (6項)：** LINE, FB Messenger, WhatsApp, Instagram, 電話 (Call), Email (Mail)。  
  * **外觀自訂 (2項)：** (1) 按鈕位置 (左下/右下)，(2) 按鈕顏色 (單色)。  
  * **交付方式：** 使用者輸入 Email，系統「寄送」程式碼至信箱。  
* **營運/後端 (Business Logic)：**  
  * **反向連結：** 在前端按鈕旁，硬編碼 (Hardcode) 恆定可見的錨點文字 報數據，連結至 https://thinkwithblack.com。  
  * **P2 數據儲存：** 在用戶點擊「寄送程式碼」時，後端**必須**將該用戶的 email 與其 config\_json (設定檔) 綁定並儲存至 P1 資料庫，為 P2 的「自動匯入」做準備。

### **4.2 P2: SaaS 升級 (建立基地)**

* **核心功能 (Features)：**  
  * **認證系統：** (B) 無密碼登入 (Magic Link)。  
  * P1 設定匯入： 登入時自動  
    SELECT config\_json FROM P1\_Users\_Data WHERE email \= ?  
  * **P2 升級誘因 (Sweetener)：** 發布 P1 沒有的「歡迎訊息 (Greeting Message)」功能，僅限 P2 會員使用。  
* **營運/後端 (Business Logic)：**  
  * **動態程式碼：** P2 的 JS 程式碼是動態的（例如 .../app.js?id=USER\_123），能即時讀取資料庫中的最新設定，用戶無需在後台修改後重新貼代碼。

### **4.3 P3: 全球擴張 (拓展領土)**

* **產品設計 (UI/UX)：**  
  * **P1 & P2 在地化：** P1 的「單頁生成器」和 P2 的「會員儀表板」都必須被完整翻譯，支援「繁中 / 日文 / 英」三語系切換。  
* **營運/後端 (Business Logic)：**  
  * **在地化反向連結 (系統升級)：** 後端系統必須升級，使其能根據「用戶註冊時的語系」動K態生成反向連結錨點文字。  
  * If (lang \== 'jp') { anchor \= 'レポートデータ' }  
  * If (lang \== 'en') { anchor \= 'Report Data' }

### **4.4 技術架構 (Technical Architecture)**

* **前端框架：** Next.js (或 React / Vue.js) \+ Tailwind CSS (易於實現 "Clone & Simplify")。  
* **後端/資料庫：** Supabase / Firebase (提供 Auth, DB, Functions，完美契合 P1 \-\> P2 升級)。  
* **Email 發送：** SendGrid (或 Amazon SES)。  
* **資料庫 Schema (P1 核心)：**  
  * **Table: P1\_Configs**  
    * id (Primary Key)  
    * email (Text, Indexed) \- 用於 P2 登入  
    * config\_json (JSONB) \- 儲存 P1 的所有設定  
    * created\_at (Timestamp)  
    * lang (Text, Default: 'zh-TW') \- 為 P3 準備

### **4.5 反向連結技術策略 (SEO Technical Spec)**

* **錨點文字：** 必須恆定可見（非 display: none）。  
* **HTML 結構 (關鍵)：**  
  * **\<a\> 標籤：** \<a href="https://thinkwithblack.com" target="\_blank" rel="noopener"\>報數據\</a\>  
  * **絕對禁止：** **嚴禁**使用 rel="nofollow" 或 rel="sponsored" 屬性，否則將導致 SEO 商業目標完全失敗。

## **5\. 里程碑與風險 (Milestones & Risks)**

### **5.1 驗證里程碑 (Validation Milestones)**

| 里程碑 | 驗證內容 | 成功指標 (KPI) |
| :---- | :---- | :---- |
| **P1 上線 14 天** | 使用者是否能成功安裝 P1 | 安裝成功率 ≥ 80% |
| **P1 上線 30 天** | P1 擴散速度 | 達成 1,000 獨立網站安裝 |
| **P2 上線 30 天** | P2 升級誘因是否有效 | P1 舊用戶升級率 ≥ 15% |
| **P3 上線 60 天** | 全球擴張是否成功 | 日/英 新用戶佔比 ≥ 30% |

### **5.2 風險管理 (Risks)**

* **風險 1 (P1)：** 使用者（學生）可能透過自訂 CSS display: none; 來隱藏 報數據 反向連結。  
  * **緩解：** 接受此風險。這是免費工具的必然成本，我們專注於服務 90% 不會這麼做的用戶。  
* **風險 2 (P2)：** P1 用戶的「P2 升級轉換率」可能很低。  
  * **緩解：** (C) 利益驅動。P2 的「歡迎訊息」功能必須做得足夠吸引人，作為強效誘因。  
* **風險 3 (P3)：** GetButton 在我們 P1 上線前，搶先推出了完美的「繁體中文版」。  
  * **緩解：** **速度優先**。P1 (MVP) 的核心是速度，必須盡快上線，搶佔「中文易用」的心智份額。  
* **風險 4 (P3)：** thinkwithblack.com 網站本身若沒有對應的日文/英文內容，來自日/英的反向連結 SEO 效益將會打折。  
  * **待辦 (to Founder)：** 需考量 P3 上線時，thinkwithblack.com 是否也需要準備對應的 .../jp 和 .../en 承接頁面。
